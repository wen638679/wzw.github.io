Maven的使用

maven常用的构造命令

  • compile 编译
  • test 测试
  • package 打包
  • clean 删除target
  • install 安装jar包到本地仓库中

maven引入构建包的流程

  1. 使用compile编译源代码
  2. 编译的过程中如果程序用到了其他的依赖包会去pom.xml查找
  3. 试图引入该依赖包的坐标
  4. 引入坐标以后，会去本地仓库中查找
  5. 如果有本地仓库中有这些jar包，会将这些jar包放到classpath中
  6. 如果没有则会去网上下载，并将起放到本地仓库中

自动创建目录骨架

使用archetype插件
用于创建符合maven规定的目录骨架
创建目录的两种方式：

1. mvn archetype:generate 按照提示进行选择
2. mvn archetype:generate -DgroupId=组织名，公司网址的反写+项目名 -DartifactId=项目名-模块名 -Dversion=版本号 -Dpackage=代码所存在的包名

pom.xml常用标签

  • modelVersion 指定了当前pom版本 4.0.0
  • version 版本号
    第一个0表示大版本号
    第二个0表示分支版本号
    第三个0表示小版本号
    snapshot 快照
    alpha 内部测试
    beta 公测
    release 稳定
    GA 正式发布
  • packaging 打包方式
    默认jar
    war zip pom
  • name 项目描述名
  • url 项目地址
  • description 项目描述
  • developers 开发人员列表
  • licenses
  • dependencies 依赖列表
    • dependency 依赖项
      • groupId
      • artifactId
        -version
      • type
      • scope 使用范围 如：test 默认为Compile
      • optional 设置依赖是否可选默认为false
      • exclusions 排除依赖传递列表
        • exclusion
          • groupId
          • artifactId
          • version
  • dependencyManagement 依赖管理
    • dependencies
      • dependency
  • build 构建
    • plugins 插件列表
      • plugin 插件
  • parent
  • modules
    • module

依赖冲突

场景：A和B都依赖C，但是依赖的版本不同，D依赖A和B，那么D会依赖那个版本的C？

  1. 短路优先
    • A > B > C > X(jar)1.0
    • A > D > X(jar) 1.1
    • A – X(jar)1.1
  2. 先声明先优先
    • 如果长度相同，则谁先声明，先解析谁
