studyspring

spring简介

  • spring是一个轻量级的控制反转（Ioc）和面向切面(AOP)的容器框架

    • 从开销和大小两方面来讲Spring都是轻量级的
    • 通过控制反转技术达到松耦合的目的
    • 提供了面向切面编程的丰富支持，允许分离应用的业务逻辑与系统级服务进行内聚性的开发
    • 包含并管理应用的对象配置和生命周期管理，在这个意义上他是一个容器
    • 将简单的组件配置,组合成为复杂的应用，这个意义上是框架

  • spring作用

    • 容器
    • 提供了多种技术的支持 JMS,MQ支持,unittest
    • AOP(事物管理，日志等)
    • 提供了众多方便应用的辅助类（JDBC Template等）
    • 对主流应用框架提供了良好的支持

IOC

接口及面向接口编程

  • 接口
    • 接口是在沟通中的中介化物
    • 实体把自己提供给外界的一种抽象化说明，用于有内部操作分离出外部沟通方法，使其能被修改内部，而不影响外界其他实体与其交互方式
  • 面向接口编程
    • 结构设计中，分清层次及调用关系，每一层只向外提供一组功能接口，各层之间只依赖接口而非实体类
    • 接口实现的变动不影响各层间的调用，这一点在公共服务中尤为重要
    • 面向接口编程中接口是用于隐藏具体实现类和实现多态性的组件

什么是IOC

  • IoC 控制反转

    • 控制权的转移，应用程序本身不负责依赖对象的创建和维护，而有外部容器创建和维护

Bean

  • Bean的配置项
  • Bean的作用域 Scope
    • singleton:单例，在一个Bean容器中只存在一份
    • prototype:每次请求（每次使用）创建新的实例，destroy方式不生效
    • request：每次http请求创建一个实例仅在当前request内有效
    • session：每次http请求创建，当前session内有效
    • global session：基于portlet的web中有效，

  • Bean的生命周期

    • 定义

    • 初始化 在启动ioc容器的时候调用

      • 实现org.springframework.beans.factory.InitializingBean接口，覆盖afterPropertiesSet方法
      • 配置init-method 配置了这个在相应的类中必须有这个方法，不然会启动ioc容器失败

      • 两种方式同时存在，接口方式的方法先与xml配置的方法调用

    • 使用

    • 销毁 ioc容器销毁的时候调用

      • 实现org.springframework.beans.factory.DisposableBean接口，覆盖destory方法
      • 配置destroy-method 配置了这个，在bean的类文件中必须有这方法

    • 配置全局默认初始化 销毁方法 配置了这个，在bean的类中不必须有这些方法

      这样里面的所有bean在初始化 销毁的时候都会调用这两个方法

  • Bean的自动装配 AutoWriting

    自动装配的四种模式 autowrite 全局配置default-autowrite

    - No 不做任何操作
- byname 根据属性名称 与 bean id进行匹配
- bytype 如果容器中存在一个与指定属性类型相同的bean，那么将与该属性自动装配；如果有多个相同类型的bean，则会报异常；如果没有找到bean则没有什么事发生
- constructor 与bytype方式类型，但是他作用在构造器参数中。如果容器中没有找到与构造器参数类型一致的bean，就会抛出异常

  • Resources&ResourceLoader

    • 针对资源文件的统一接口
    • Resources
      • UrlResource:URL对应的资源，根据一个URL地址即可构建
      • ClassPathResource:获取类路径下的资源文件
      • FileSystemResource：获取文件系统里的资源文件
      • ServletContextResource：ServletContext封装的资源，用于访问ServletContext环境下的资源
      • InputStreamResource：针对输入流封装的资源
      • ByteArrayResource：针对字节数组封装的资源

    • ResourceLoader

      • 对Resources进行加载的类

        所有application context都实现了ResourceLoader接口

      • prefix
        • classpath: classpath:come/myapp/config.xml
        • file: file:/data/config.xml
        • http: url:http://myServlet/logo.png
        • none: /data/config.xml

Aware

  • spring中提供了一些以Aware结尾的接口，实现了Aware接口的bean在被初始化之后可以获得相应的资源
  • 通过Aware接口，可以对spring的相应资源进行操作
  • 为对spring进行简单的扩展提供了方便的入口
    • ApplicationContextAware
    • BeanNameAware

Bean管理的注解实现

classpath扫描与组件管理

  • 使用java注解的方式而不是xml的形式来定义Bean，如：@Configuration， @Bean， @Import， @DependsOn
  • @Component是一个通用注解，可用于任何Bean
  • @Repository， @Service， @Controller是更具有针对性的注解
    • @Repository通常用于注解DAO类， 即持久层
    • @Service通常用于注解Service类，即服务层
    • @Controller通常用于注解Controller类，即控制层
